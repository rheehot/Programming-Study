## 동기/비동기
> 순서보장 매커니즘

`동기` : 현재 작업의 요청과 응답이 동시에 발생하는것 -> 현재 작업의 응답과 다음 작업의 요청이 동시에 발생<br/><br/>
`비동기` : 동기의 반대 상태 -> 현재 작업의 응답과 다음 작업의 요청의 타이밍이 일치하지 않아도 됨
